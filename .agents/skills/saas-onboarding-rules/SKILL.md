# SaaS Onboarding Rules

## Goal
Tornar a primeira experiência da nutricionista assinante rápida, guiada e comercialmente convincente, gerando um "AHA Moment" nos primeiros 3 minutos de uso.

## Instructions
Quando estiver atuando no onboarding, telas iniciais ou estados vazios, você deve:
1. Avaliar a tela para criar modais úteis ou banners que guiam para a próxima ação de valor ("adicione um paciente vivo ou simule a IA primeiro").
2. Prevenir que o dashboard seja apenas uma imensidão de cards brancos com métricas "zero". Preencha com CTAs proeminentes.
3. Projetar passos iniciais que capturem ou mostrem o Branding do cliente (nome, logo, cores) imediatamente.
4. Mapear o funil: Login -> Setup de Perfil -> 1º Paciente -> 1ª Consulta -> 1ª Dieta AI -> 1º Export/PDF. Reduza qualquer passo mecânico no meio disso.

## Examples
- Substituir a lista vazia de "Pacientes sem retorno" por um bloco: "Nenhum paciente ainda? \n [ + Cadastrar meu primeiro paciente ]".
- O Modal de "Bem-vinda" com introdução sobre o Assistente de Inteligência Artificial usando GPT-4.

## Constraints
- Nunca alongar onboarding com mais de 3 passos sem recompensa.
- Nunca esconder o Call To Action (botão principal) ou diluí-lo com botões menores.
- Não transformar o UX em um tutorial invasivo travando a tela do usuário.
